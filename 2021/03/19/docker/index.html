容器镜像的导出导入

docker 容器 2021/03/19

容器基础

作为系统运维，不可避免的遇到容器相关问题，所以了解容器的一些基本命令也是基本素养。
介绍几个基本的关于，运维场景中所需要的操作命令。

一、容器和镜像的导出何导入操作。

在没有hub场景或者网络情况下，对容器何镜像的操作，只有回归最原始的文件模式，即：导出导入的基本方法。
类型 命令 用途
镜像操作 docker images 列出本地容器镜像
镜像操作 docker save -o test.tar image_name 打包备份镜像为文件
镜像操作 docker load -i(–input) /test.tar 从镜像文件导入镜像
容器操作 docker export [-o –output[=””]] CONTATINER
容器操作 docker export 容器id/name >文件名.tar 容器导出 范例
容器操作 docker export -o test.img test 容器导出 范例
容器操作 docker import -o test.img test 用来载入容器包，导入后是镜像

容器和镜像导出导入对比

操作 类别 备注
存镜像 docker save 导出镜像 image
存容器 docker export 导出镜像 保存的是容器（container）
载入镜像 docker load 用来载入镜像包，image 恢复为镜像
载入容器包 docker import 用来载入容器包， 恢复为镜像 
 容器和镜像操作对比，导出导入对比。docker save保存的是镜像（image），docker export 保存的是容器（container）；用来载入镜像包，docker import 用来载入容器包，但两者都会恢复为镜像；docker load不能对载入的镜像重命名，而docker import可以为镜像指定新名称。

二、 容器的启动何进入：
1 启动：
docker run -d –name container_test –privileged -it test:latest /usr/sbin/init
2 进入镜像：
docker exec -it container_test /bin/bash
