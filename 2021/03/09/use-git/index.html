二、git命令介绍：

我自己写了点git使用命令，基本满足运维需求。
之所以选择命令操作，因为工具各异，不好统一。
1 命令是运维基本需要的，运维大多在服务器上调试，基本上没什么界面。
2 工具各异本质都是命令，修复问题，命令也是最后的大招了。

操作命令 说明备注 范例 备注
git checkout -b name name 创建的分支名称 git checkout -b dev 创建dev分支，并切到新分支 ‘dev’
git branch 查看分支 列出分支清单并且 * 表示当前所在分支
git add fname fname 文件名 git add readme.txt 添加readme.txt 到git版本管理里
git commit 提交文件 git commit -m “add readme.txt dev” 提交文件到仓库
git checkout name 切换到分支name git checkout master 切换到master分支
git merge name 合并分支 name git merge dev 把分支dev 合并到当前分支
git tag … 这个一般为了发布版本 包含创建 删除等操作
